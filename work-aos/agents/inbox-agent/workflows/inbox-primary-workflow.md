# Inbox Primary Workflow

Primary workflow for the Inbox / Communications Agent.

1. Triage incoming items and summarize what needs attention.
2. Decide each item's destination: task, project, calendar, memory, decision, or archive.
3. Draft any needed replies (sending is gated by `Proceed`).
4. Move processed source items to `/inbox/processed`.
5. Log promotion and triage decisions.
